# 改进方案说明 · JD Gap Analyzer

> **作者**：liuzh  
> **改进对象**：SpeedyJob（speedyjob.cn）JD 匹配功能  
> **技术栈**：Python · Streamlit · DeepSeek API  

---

## 一、选择改进的功能

SpeedyJob 的 **JD 匹配分析**功能。

该功能现有形态：上传简历后输入目标岗位，AI 给出一个综合匹配分数（0-100）和若干条泛化建议，如"建议补充项目经验"、"可以强化技术描述"。

---

## 二、当前问题

| 问题 | 具体表现 |
|------|----------|
| **诊断颗粒度太粗** | 只输出整体分数，无法知道"哪一条 JD 要求没达到" |
| **建议缺乏可操作性** | 给出"加强项目描述"这类模糊建议，求职者不知道如何修改 |
| **缺少差距定位** | 没有指出简历中已有哪些内容对应了 JD，哪些是盲区 |
| **无改写示范** | 即使发现问题，也没有给出可直接使用的改写文字 |

**核心矛盾**：求职者花时间看完报告，依然不知道下一步具体改什么。

---

## 三、改进方案

**JD Gap Analyzer —— 逐条拆解 · 精准定位 · 即时改写**

### 设计思路

将 JD 中每一条要求拆解为独立的分析单元，对每条分别判断覆盖程度，并给出具体建议。

```
JD 原文要求
    ↓
逐条分析（hit / weak / gap）
    ↓
定位简历中的对应证据
    ↓
针对弱项/缺失项，生成可直接粘贴的改写句子
```

### 三级覆盖评估

| 状态 | 含义 | 处理 |
|------|------|------|
| ✅ hit（覆盖） | 简历中有明确对应内容 | 展示证据，无需修改 |
| ⚠️ weak（较弱） | 有相关内容但描述不够有力 | 给出强化改写建议 |
| ❌ gap（缺失） | 简历中完全没有相关内容 | 给出补充方向或可用语句 |

### 输出结构

1. **总览仪表板**：匹配分、hit/weak/gap 各项计数
2. **逐条分析卡片**：每条 JD 要求 + 状态标签 + 简历证据 + 改写建议
3. **优势亮点**：简历相对该 JD 的差异化优势
4. **行动清单**：按优先级排列的3个最高价值行动

---

## 四、为什么这样设计

**从"读完不知道改什么"到"读完直接能改"**

- **逐条对应**：JD 是 HR 的筛选清单，逐条对齐才能真正指导修改，而非整体评估
- **三级状态而非二元**：weak 状态最有价值——这些是"已有基础、小改即有收益"的快速提升点
- **直接给改写文字**：求职者面临时间压力，"给方向"不如"给句子"，可直接采用或微调
- **技术选型轻量**：Streamlit 无需部署后端，API Key 本地使用，隐私友好，5分钟可本地跑起来

## 五、如何用 Vibe Coding 完成整个过程

本项目全程使用 AI 辅助开发，以下是实际协作过程：

**① 需求拆解（和 AI 对话确定方向）**

直接向 AI 描述目标："SpeedyJob 的 JD 匹配只给分数，能不能做一个逐条分析版本？"AI 帮我分析了竞品现状、识别出"颗粒度不足"这个核心痛点，并建议了三级状态（hit/weak/gap）的分层设计——这比我最初想的二元判断要合理得多。

**② 方案设计（AI 辅助结构化）**

AI 帮我设计了 JSON 输出格式（system prompt + 结构化输出），确保了 AI 返回的分析结果可以被程序直接解析和渲染，避免了非结构化输出带来的解析麻烦。

**③ 代码生成（AI 写初版，我调细节）**

AI 生成了完整的 Streamlit 应用骨架：输入区布局、API 调用、JSON 解析、卡片渲染。我主要做了两处调整：
- 加入正则提取 JSON（`re.search(r'\{.*\}', raw, re.DOTALL)`），防止模型输出 Markdown 代码块包裹导致 JSON 解析失败
- 调整 CSS 颜色系统，统一用 hit/weak/gap 三色（绿/黄/红）贯穿所有 UI 组件

**④ 调试与迭代（AI 帮定位问题）**

遇到"示例数据加载后无法刷新"的 Streamlit 状态问题，AI 给出了 `st.session_state` + `st.rerun()` 的标准解法；卡片 HTML 渲染的样式问题，AI 帮我调整了 flexbox 对齐方式。

**结论**：Vibe Coding 的核心价值不是"替代思考"，而是**加速从想法到可运行代码的速度**。整个 MVP 从 0 到跑通大约用了 1.5 小时，其中约 70% 的时间在想清楚"要做什么"，30% 在让 AI 实现它。

## 六、快速启动

```bash
# 1. 安装依赖
pip install streamlit openai

# 2. 运行
streamlit run app.py

# 3. 在侧边栏输入 DeepSeek API Key，粘贴简历和JD，点击分析
```

> DeepSeek API：https://platform.deepseek.com  
> 调用成本：每次分析约 ¥0.01（deepseek-chat 模型）
